# 第2章：カーネル化

> カーネル化は、多項式時間前処理アルゴリズムを体系的に研究するためのアプローチである。パラメータ化アルゴリズムの設計において重要なツールである。本章では、クラウン分解、拡張補題、ひまわり補題、線形計画法といった基本的なカーネル化技法を解説する。これらの技法を用いて、頂点被覆、トーナメントにおけるフィードバック弧集合、辺クリーク被覆、最大充足可能性、およびd-ヒッティング集合に対するカーネルを得ることで、これらの技法を具体的に示す。

前処理（データ削減またはカーネル化）は、NP困難問題を扱うことを目的としたほぼすべての実用的なコンピュータ実装において普遍的に使用されている。前処理サブルーチンの目的は、問題インスタンスの容易な部分を効率的に解き、計算上困難な核心構造（インスタンスの問題カーネル）にまで縮小（縮小化）することである。言い換えれば、この手法の考え方は、与えられた問題インスタンスを、入力サイズの多項式時間で、同値なより小さいサイズのインスタンスに縮小する（ただし必ずしも解くわけではない）というものである。その後、この小さくなったインスタンスに対して、より遅い厳密アルゴリズムを実行することができる。

このような前処理サブルーチンの有効性をどのように測定できるだろうか？有用な前処理アルゴリズムを、多項式時間で実行され、インスタンス I を少なくとも1ビット小さい同値なインスタンスに置き換えるものと定義したとしよう。するとNP困難問題に対するそのようなアルゴリズムの存在は P = NP を意味することになり、そのようなアルゴリズムが見つかる可能性は低い。長い間、有用な前処理の形式的定義として他に提案がなく、多項式時間前処理アルゴリズムの数学的解析はほとんど顧みられなかった。しかし、パラメータ化計算量の言語を用いれば、パラメータが小さい大きなインスタンスは縮小されるべきであり、一方でパラメータに比べて小さいインスタンスはさらに処理する必要がないと要求することで、有用な前処理の定義を定式化できる。これらの考え方が、カーネル化と呼ばれる多項式時間アルゴリズムの失われた大陸を切り開く。

本章では、具体的な例を通じて、カーネル化アルゴリズムを設計するためによく使われるいくつかの技法を解説する。次の節、第2.1節では、形式的な定義を与える。第2.2節では、いわゆる自然な帰着規則に基づくカーネル化アルゴリズムを示す。第2.3節では、クラウン分解と拡張補題の概念を導入し、最大充足可能性問題への適用を示す。第2.5節では、線形計画法に基づくツールを検討し、頂点被覆に対するカーネルを与える。最後に、第2.6節でひまわり補題を学び、これを用いてd-ヒッティング集合に対する多項式カーネルを得る。

## 2.1 形式的定義

ここで、カーネル化の概念を捉える形式的定義に移る。パラメータ化問題 Q に対するデータ削減規則、あるいは単に帰着規則とは、関数 φ: Σ∗×N → Σ∗×N であって、Q のインスタンス (I, k) を Q の同値なインスタンス (I′, k′) に写すものであり、φ は |I| と k の多項式時間で計算可能である。Q の2つのインスタンスが同値であるとは、(I, k) ∈ Q であることと (I′, k′) ∈ Q であることが同値であることを言う。帰着規則 φ のこの性質、すなわちインスタンスを同値なインスタンスに変換するという性質は、帰着規則の安全性（safeness）または健全性（soundness）と呼ばれることがある。本書では、規則は安全（safe）である、および帰着規則の安全性（safeness）という表現を用いる。

一般的な考え方は、インスタンスサイズをできる限り縮小するために、さまざまなデータ削減規則を逐次適用する前処理アルゴリズムを設計することである。したがって、そのような前処理アルゴリズムは、Q のインスタンス (I, k) ∈ Σ∗×N を入力として受け取り、多項式時間で動作し、Q の同値なインスタンス (I′, k′) を返す。出力インスタンスが小さくなければならないという要求を形式化するために、パラメータ化計算量の主原理を適用する：計算量はパラメータの関数として測定される。したがって、前処理アルゴリズム A の出力サイズは、関数 sizeA: N → N ∪{∞} として以下のように定義される：

sizeA(k) = sup{|I′| + k′ : (I′, k′) = A(I, k), I ∈ Σ∗}

すなわち、固定されたパラメータ k を持つ Q のすべての可能なインスタンスを考え、これらのインスタンスに対する A の出力サイズの上限を測定する。この上限は無限大になりうることに注意されたい。これは、入力パラメータ k のみの関数として A(I, k) のサイズに上界がない場合に起こる。カーネル化アルゴリズムとは、まさにその出力サイズが有限であり、パラメータの計算可能な関数で上から抑えられる前処理アルゴリズムのことである。

**定義 2.1（カーネル化、カーネル）。** パラメータ化問題 Q に対するカーネル化アルゴリズム、あるいは単にカーネルとは、Q のインスタンス (I, k) が与えられたとき、多項式時間で動作し、Q の同値なインスタンス (I′, k′) を返すアルゴリズム A のことである。さらに、ある計算可能な関数 g: N → N に対して sizeA(k) ≤ g(k) が成り立つことを要求する。

この定義におけるサイズの要求は次のように言い換えられる：ある計算可能な関数 g(·) が存在して、(I′, k′) がインスタンス (I, k) に対する出力であるときは常に、|I′| + k′ ≤ g(k) が成り立つ。上界 g(·) がパラメータの多項式（線形）関数であるとき、Q は多項式（線形）カーネルを許容するという。表記を濫用して、カーネル化アルゴリズムの出力である縮小された同値なインスタンスもカーネルと呼ぶことが多い。

本章の中で、ある境界的な場合に、考慮している問題インスタンスを完全に解決できる、すなわち、それがyesインスタンスであるかnoインスタンスであるかを正しく判定できる状況にしばしば遭遇する。したがって、明確にするために、帰着（そしてその結果としてカーネル化アルゴリズム）が、縮小されたインスタンスの代わりにyes/noの答えを返すことを許容する。形式的には、導入したカーネルの定義に適合させるため、そのような場合にカーネル化アルゴリズムは定数サイズの自明なyesインスタンスまたはnoインスタンスを返すべきである。このようなインスタンスは、空集合とその補集合を除くすべてのパラメータ化言語に対して存在し、したがってカーネル化アルゴリズムにハードコードすることができることに注意されたい。

Q のインスタンス (I, k) が与えられたとき、カーネルのサイズは、縮小された同値なインスタンス I′ を符号化するのに必要なビット数にパラメータ値 k′ を加えたものとして定義されることを思い出されたい。しかし、グラフ、ハイパーグラフ、または論理式に関する問題を扱う際には、出力インスタンスの他の側面を強調したいことが多い。例えば、グラフ問題 Q に対して、出力インスタンスの頂点数と辺数の上界を強調するために、Q は O(k³) 個の頂点と O(k⁵) 本の辺を持つカーネルを許容する、と言うことがある。同様に、論理式上で定義された問題に対して、その問題は O(k) 個の変数を持つカーネルを許容する、と言うことがある。

ここで重要なのは、カーネル化の初期の定義では k′ ≤ k が要求されていたことを言及しておくことである。直感的には、パラメータ k が問題の複雑さを測定するものであり、k が大きいほど問題は困難であるから、これは理にかなっている。この要求は、特に下界の文脈において、後に緩和された。より自由なカーネル化の概念の利点は、カーネルの多項式変換に対して頑健であることである。しかし、実用的な前処理との関連は制限される。本章で言及するすべてのカーネルは、出力パラメータが入力パラメータ以下である、すなわち k′ ≤ k であるという条件を満たしている。

通常、計算機科学ではアルゴリズムの効率を実行時間の見積もりによって測定するが、カーネル化アルゴリズムの効率の中心的な尺度は出力サイズの上界である。カーネル化アルゴリズムの実際の実行時間は実用的な応用においてしばしば非常に重要であるが、理論的にはカーネル化アルゴリズムは多項式時間で動作することのみが要求される。

ある問題に対してカーネル化アルゴリズムがあり、さらに (I, k) がyesインスタンスであるかどうかを判定する（実行時間は問わない）何らかのアルゴリズムがあれば、縮小されたインスタンス I のサイズは単に k の関数（入力サイズ n とは独立）であるから、明らかにその問題はFPTである。しかし、驚くべきことに、逆もまた成り立つ。

**補題 2.2.** パラメータ化問題 Q がFPTであるならば、Q はカーネル化アルゴリズムを許容する。

証明. Q がFPTであるから、ある計算可能な関数 f と定数 c に対して、(I, k) ∈ Q であるかどうかを時間 f(k)·|I|^c で判定するアルゴリズム A が存在する。Q に対するカーネル化アルゴリズムを次のようにして得る。入力 (I, k) が与えられたとき、カーネル化アルゴリズムは (I, k) に対して A を最大 |I|^{c+1} ステップだけ実行する。もし答えが得られて終了すれば、その答えを用いて (I, k) がyesインスタンスであるかnoインスタンスであるかを返す。もし A が |I|^{c+1} ステップ以内に終了しなければ、(I, k) 自体をカーネル化アルゴリズムの出力として返す。A が |I|^{c+1} ステップで終了しなかったので、f(k)·|I|^c > |I|^{c+1} が成り立ち、したがって |I| < f(k) である。その結果、|I| + k ≤ f(k) + k となり、サイズが最大 f(k) + k のカーネルが得られる。なお、f(k) が計算可能な関数であるから、この上界は計算可能であることに注意されたい。

補題 2.2は、決定可能な問題がカーネルを許容するのは、それが固定パラメータ易処理（FPT）であるときに限ることを意味する。したがって、ある意味で、カーネル化は固定パラメータ易処理性を定義するもう一つの方法であると言える。

しかし、この理論的結果から得られるカーネルは通常、指数的な（あるいはさらに悪い）サイズであるのに対し、問題固有のデータ削減規則はしばしば二次（g(k) = O(k²)）または線形サイズ（g(k) = O(k)）のカーネルを達成する。したがって、任意の具体的なFPT問題に対する自然な問いは、パラメータの多項式関数で上から抑えられる問題カーネル（g(k) = k^{O(1)}）を許容するかどうかということである。本章では、いくつかの初等的な手法を用いて、いくつかの問題に対する多項式カーネルを与える。第9章では、カーネルを得るためのより進んだ手法を与える。
